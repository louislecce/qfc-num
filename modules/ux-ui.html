UX vs UI : quelle différence ?

UX (User Experience) désigne l'expérience globale d'un utilisateur avec un produit : est-ce qu'il comprend comment l'utiliser ? Atteint-il son objectif facilement ? Est-il satisfait ?

UI (User Interface) désigne la couche visuelle et interactive : les boutons, les couleurs, la typographie, les icônes. L'UI sert l'UX — un beau design qui désoriente est mauvais.

💡

Analogie : L'UX est l'architecture d'un bâtiment (flux de circulation, logique des pièces), l'UI est la décoration intérieure (couleurs, matériaux, mobilier).

Concepts clés à maîtriser

🗺️

User Flow

Le chemin que parcourt un utilisateur pour accomplir une tâche. Ex : s'inscrire → paramétrer son profil → faire sa première action.

📐

Hiérarchie visuelle

Organiser les éléments pour guider l'œil : taille, couleur, contraste, espacement. L'utilisateur doit savoir quoi regarder en premier.

🖐️

Affordance

La capacité d'un élément à suggérer son utilisation. Un bouton doit "ressembler à un bouton" pour être cliqué sans hésitation.

♿

Accessibilité

Concevoir pour tous les utilisateurs, y compris ceux avec des handicaps visuels, moteurs ou cognitifs. C'est aussi un critère légal.

Les erreurs classiques à éviter

  • Concevoir pour soi plutôt que pour l'utilisateur (le "biais du concepteur")
  • Trop de fonctionnalités sur un seul écran — surcharger l'interface nuit à la compréhension
  • Négliger les états vides, de chargement et d'erreur — ce sont des moments clés de l'expérience
  • Ignorer la cohérence : même composant = même comportement sur toute l'application
  • Penser mobile en dernier — le mobile doit souvent être le point de départ
