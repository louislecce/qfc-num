Parler tech sans être développeur

Un fondateur non-technique n'a pas besoin de savoir coder — mais il doit maîtriser le vocabulaire pour dialoguer efficacement avec son équipe tech, comprendre les arbitrages et ne pas se faire embarquer dans de mauvaises décisions architecturales.

Ce module est un glossaire pratique organisé par couches : du visible (frontend) à l'invisible (infrastructure), en passant par ce qui relie les deux (API, backend).

🔑

Principe clé : Tu n'as pas besoin de comprendre comment ça marche en détail — tu dois comprendre pourquoi on choisit telle ou telle approche et quelles en sont les conséquences pour le produit.

Les couches d'une application

🖥️

Frontend

Ce que l'utilisateur voit et avec quoi il interagit. HTML, CSS, JavaScript, React, Vue... S'exécute dans le navigateur ou sur l'appareil.

⚙️

Backend

Le moteur invisible : logique métier, traitement des données, authentification. Node.js, Python, Ruby... S'exécute côté serveur.

🔌

API

Interface qui permet à deux applications de communiquer. Le frontend appelle l'API pour demander des données au backend.

🗄️

Base de données

Stockage structuré des données. SQL (relationnel, structuré) vs NoSQL (flexible, scalable). Le choix impacte l'architecture entière.

☁️

Cloud & Infrastructure

AWS, GCP, Azure : serveurs à la demande, scalables. Évite d'acheter du matériel. On paye à l'usage.

🚀

No-code / Low-code

Outils permettant de créer des apps sans coder (Webflow, Bubble, Make). Idéal pour valider rapidement avant de développer.

Web, mobile, natif : quelles différences ?

  • Application web : accessible via navigateur, pas d'installation, mise à jour immédiate — le choix par défaut pour démarrer
  • Application mobile native : développée pour iOS (Swift) ou Android (Kotlin), distribuée via stores — performances optimales mais coût x2
  • Application hybride (React Native, Flutter) : un seul code pour iOS et Android — bon compromis coût/performance pour les startups
  • PWA (Progressive Web App) : site web avec comportement d'app mobile — installable, hors-ligne, sans passer par les stores
💡

Conseil startup : Commence toujours par le web. C'est plus rapide à développer, plus facile à itérer et tu évites les contraintes des stores Apple et Google dès le départ.
